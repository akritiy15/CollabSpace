API Documentation

CollabSpace v1 REST API

Authentication Requirement
All endpoints (except login) require an Authorization header: Authorization: Bearer <your_access_token> Get your token from POST /api/v1/auth/login.

Authentication

POST /api/v1/auth/login

Authenticate user with email and password to receive JWT tokens.

Request Body

{
  "email": "user@example.com",
  "password": "password123"
}

Success Response

{
  "success": true,
  "data": {
    "access_token": "eyJhb...",
    "refresh_token": "eyJhb...",
    "user": {
      "id": 1,
      "username": "johndoe",
      "email": "user@example.com"
    }
  },
  "error": null
}
POST /api/v1/auth/refresh
Refresh Token Required

Exchange a valid refresh token for a new access token.

Success Response

{
  "success": true,
  "data": { "access_token": "new_eyJhb..." },
  "error": null
}

Groups

GET /api/v1/groups/
Auth Required

Returns all groups the current user is a member of.

Success Response

{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Project Alpha",
      "member_count": 5,
      "task_count": 12,
      "user_role": "admin"
    }
  ],
  "error": null
}
POST /api/v1/groups/
Auth Required

Create a new group. The creator automatically becomes an admin.

Request Body

{
  "name": "New Marketing Team",
  "description": "Team collaboration space"
}

Tasks

GET /api/v1/groups/:group_id/tasks/
Auth Required

Get all tasks for a specific group. User must be a group member.

Query Parameters

  • ?status=todo | in_progress | done - Filter by status
  • ?assigned_to=me - Filter tasks assigned to current user
  • ?sort=newest | oldest | deadline - Sort order
PATCH /api/v1/groups/:group_id/tasks/:task_id/
Editor/Admin Required

Update an existing task. Only include fields you want to change.

Request Body Example

{
  "status": "done",
  "assigned_to": 3
}

Reports & Activity

GET /api/v1/groups/:group_id/report/
Admin Required

Returns full analytics and breakdown report natively mirroring the PDF specification payload in strict JSON structure.

Response Structure

{
  "success": true,
  "data": {
    "group": { "id": 1, "name": "Team" },
    "summary": { "total_tasks": 10, "completion_rate": 80 },
    "tasks": [...],
    "members": [...],

    "recent_activity": [...]
  },
  "error": null
}
GET /api/v1/groups/:group_id/activity/
Auth Required

Retrieve a paginated history feed mapped strictly per group activity boundaries.

Query Parameters

  • ?page=1 - Native pagination cursor
  • ?per_page=20 - Items returned bounded safely
  • ?filter=task | members
